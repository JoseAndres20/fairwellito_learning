| **Nomenclatura**    | **Nombre Completo**                                         | **¿Qué es?**                                                         | **¿Para qué se usa?**                                                                            |
| ------------------- | ----------------------------------------------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------ |
| **OSSTMM**          | Open Source Security Testing Methodology Manual             | Un marco de trabajo científico y centrado en métricas.               | Para realizar evaluaciones de seguridad cuantificables y auditables mediante métricas RAV.       |
| **OWASP**           | Open Worldwide Application Security Project                 | Una fundación abierta que establece estándares y guías de seguridad. | Para estandarizar las mejores prácticas y metodologías de seguridad en aplicaciones.             |
| **WSTG**            | Web Security Testing Guide                                  | La guía de referencia de OWASP para aplicaciones web.                | Para probar aplicaciones web mediante más de 90 casos de prueba estructurados.                   |
| **MASTG**           | Mobile Application Security Testing Guide                   | El marco especializado de OWASP para seguridad móvil.                | Para probar específicamente la seguridad en aplicaciones móviles Android e iOS.                  |
| **NIST SP 800-115** | NIST Special Publication 800-115                            | Una guía técnica para evaluaciones de seguridad.                     | Para realizar evaluaciones en entornos gubernamentales y empresariales con flexibilidad técnica. |
| **PTES**            | Penetration Testing Execution Standard                      | Un marco de trabajo práctico basado en fases.                        | Para estructurar un pentest completo, desde la planificación hasta el informe final.             |
| **ISSAF**           | Information Systems Security Assessment Framework           | Un modelo de progresión adversaria de nueve pasos.                   | Se utiliza actualmente como referencia educativa para entender metodologías de ataque.           |
| **MITRE ATT&CK**    | MITRE Adversarial Tactics, Techniques, and Common Knowledge | Un catálogo de comportamientos y tácticas adversarias.               | Para usar un lenguaje universal al describir amenazas y enriquecer hallazgos con IDs reales.     |
| **PCI DSS**         | Payment Card Industry Data Security Standard                | Un mandato regulatorio obligatorio para la industria.                | Para asegurar entornos que manejan datos de tarjetas de pago.                                    |
| **CBEST**           | CBEST Intelligence-Led Testing                              | Un marco de trabajo regulado basado en inteligencia.                 | Para pruebas en instituciones financieras del Reino Unido.                                       |
| **CSA CCM**         | Cloud Security Alliance Cloud Controls Matrix               | Un conjunto de controles de gobernanza para la nube.                 | Para evaluar la postura de seguridad y controles en entornos cloud.                              |
| **WASC TC**         | Web Application Security Consortium Threat Classification   | Una taxonomía histórica de amenazas web.                             | Se utiliza como referencia de legado.                                                            |
